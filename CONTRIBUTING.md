# Contribuir a Página ACA

## Ramas

- `main`: versión estable.
- `develop`: integración, si el equipo decide utilizarla.
- `feat/<descripcion>`: nuevas funcionalidades.
- `fix/<descripcion>`: correcciones.
- `docs/<descripcion>`: documentación.
- `chore/<descripcion>`: mantenimiento técnico.

## Antes de implementar

1. Confirmar que existe una especificación.
2. Revisar preguntas abiertas.
3. Identificar criterios de aceptación.
4. Dividir el trabajo en un cambio pequeño y revisable.

## Pull requests

Todo pull request debe indicar:

- Especificación relacionada.
- Qué cambia.
- Qué no cambia.
- Cómo se probó.
- Capturas cuando afecta interfaz.
- Riesgos, migraciones o decisiones pendientes.

## Commits sugeridos

Se recomienda Conventional Commits:

- `feat: add responsive public navigation`
- `fix: prevent horizontal overflow on mobile`
- `docs: define member credential rules`
- `test: cover credential verification states`
